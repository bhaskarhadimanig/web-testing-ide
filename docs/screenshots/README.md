## Creating Screenshots

1. Run the application: `pnpm dev`
2. Use a consistent browser size: 1280x720
3. Use consistent IDE window size: 1200x800
4. Capture screenshots at 2x resolution for clarity
5. Save as PNG format with descriptive names
6. Include callouts and highlights where helpful

## Tools Recommended
- **macOS**: Screenshot utility (Cmd+Shift+4)
- **Windows**: Snipping Tool or Snip & Sketch
- **Linux**: GNOME Screenshot or KDE Spectacle
- **Cross-platform**: Flameshot, LightShot

## Guidelines
- Keep screenshots up-to-date with UI changes
- Use consistent styling and highlighting
- Focus on the specific feature being demonstrated
- Avoid including sensitive or personal information
- Use annotations to highlight important UI elements
- Maintain consistent window sizes across screenshots

## Screenshot Specifications
- **Format**: PNG with transparency support
- **Resolution**: Minimum 1200x800, preferably 2400x1600 (2x)
- **Color Depth**: 24-bit color
- **Compression**: Optimized for web delivery
- **File Size**: Keep under 500KB per screenshot

## Annotation Guidelines
- Use red arrows or circles to highlight interactive elements
- Add text callouts for important features
- Use consistent color scheme for annotations
- Keep annotations minimal and focused
- Ensure text is readable at different zoom levels

## Maintenance
- Update screenshots when UI changes significantly
- Review screenshots quarterly for accuracy
- Test screenshots on different screen sizes
- Validate that all referenced features still exist
- Archive outdated screenshots with version tags
