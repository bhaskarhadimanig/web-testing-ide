# Privacy Policy

## Overview

The Web Testing IDE is designed with privacy as a fundamental principle. This document explains our commitment to protecting your privacy and data.

## Core Privacy Principles

### Zero Data Collection
- **No Personal Information**: We do not collect, store, or process any personal information
- **No Usage Analytics**: No tracking of how you use the application
- **No Telemetry**: No automatic reporting of errors, crashes, or usage patterns
- **No Cookies**: No web cookies or tracking technologies

### Local-Only Operation
- **Offline-First**: Application works completely offline
- **No External Servers**: No communication with external servers or APIs
- **No Cloud Storage**: All data remains on your local machine
- **No Account Required**: No user accounts, registration, or authentication

## Data Storage

### What Data is Stored Locally
- **Test Recordings**: User-recorded browser interactions
- **Generated Code**: Test code generated from recordings
- **Application Settings**: User preferences and configuration
- **Test Results**: Execution results, screenshots, and reports

### Where Data is Stored
- **SQLite Database**: Local database file on your machine
- **File System**: Screenshots, recordings, and reports in local directories
- **OS Keychain**: Secure credential storage (optional, user-controlled)

### Data Retention
- **User-Controlled**: You control all data retention and deletion
- **No Automatic Cleanup**: Data persists until you manually delete it
- **Complete Removal**: Uninstalling the application removes all associated data

## Network Activity

### What Network Requests Are Made
- **Test Target Websites Only**: Connections only to websites you explicitly choose to test
- **No Background Requests**: No automatic network activity
- **No Update Checks**: No automatic checking for application updates
- **No Error Reporting**: No automatic error or crash reporting

### Browser Extension (Optional)
- **Local Communication Only**: Extension communicates only with local application
- **No External Requests**: Extension makes no external network requests
- **User-Controlled**: Extension is completely optional and user-installed

## Third-Party Services

### No Third-Party Integrations
- **No Analytics Services**: No Google Analytics, Mixpanel, or similar services
- **No Error Tracking**: No Sentry, Bugsnag, or similar error tracking
- **No CDNs**: No external content delivery networks
- **No Social Media**: No social media integrations or sharing

## Open Source Transparency

### Code Visibility
- **Open Source**: All source code is publicly available on GitHub
- **Auditable**: Anyone can review the code for privacy compliance
- **Community-Driven**: Privacy practices are community-verified
- **No Hidden Code**: No proprietary or closed-source components

## Your Rights

### Data Control
- **Full Ownership**: You own all data created by the application
- **Export Capability**: Export your data in standard formats
- **Delete Capability**: Delete any or all data at any time
- **No Vendor Lock-in**: Data is stored in open, standard formats

### Privacy Choices
- **Opt-Out by Default**: No data collection to opt out of
- **Granular Control**: Control exactly what data is stored and where
- **No Forced Updates**: Choose when and if to update the application

## Technical Implementation

### Privacy by Design
- **Minimal Data Collection**: Only collect data necessary for functionality
- **Data Minimization**: Store only essential information
- **Purpose Limitation**: Data used only for intended purposes
- **Storage Limitation**: No indefinite data retention

### Security Measures
- **Local Encryption**: Sensitive data encrypted at rest
- **Secure Communication**: All IPC communication is encrypted
- **Access Controls**: Strict file system permissions
- **Regular Audits**: Ongoing privacy compliance reviews

## Compliance

### Regulatory Compliance
- **GDPR Compliant**: No personal data processing means automatic compliance
- **CCPA Compliant**: No data sale or sharing
- **COPPA Compliant**: No data collection from users of any age
- **Enterprise Ready**: Suitable for use in regulated industries

### International Standards
- **ISO 27001**: Information security management principles
- **SOC 2**: Security, availability, and confidentiality controls
- **Privacy Shield**: Data protection framework compliance

## Data Subject Rights

### Under GDPR (Even Though We Don't Collect Data)
- **Right to Access**: View any data we might have (none)
- **Right to Rectification**: Correct any inaccurate data (none to correct)
- **Right to Erasure**: Delete any data (already local-only)
- **Right to Portability**: Export data in standard formats (already available)

### Under CCPA
- **Right to Know**: What data is collected (none)
- **Right to Delete**: Remove personal information (none collected)
- **Right to Opt-Out**: Stop data sale (no data to sell)
- **Right to Non-Discrimination**: Equal service regardless of privacy choices

## Children's Privacy

- **No Age Verification**: No collection of age information
- **No Child Data**: No special handling needed as no data is collected
- **Parental Controls**: Parents can monitor local application usage
- **Educational Use**: Safe for use in educational environments
